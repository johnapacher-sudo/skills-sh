# tech-tutorial 技能说明

## 概述

本技能的目的是基于前期生成的技术发现报告（01-discovery.md）和技术分析报告（02-analysis.md），为用户生成一份从零到一的完整硬核学习教程。教程必须涵盖环境搭建、入门知识、进阶特性、高级用法、实战项目以及常见问题排查，确保读者能够按照教程从零基础逐步掌握目标技术的核心能力。教程的每一节都应当有完整可运行的代码示例、清晰的讲解和配套练习题，使读者在阅读过程中能够边学边练，真正掌握技术细节。

## 前置条件

在开始生成教程之前，必须确保以下前置文件已经存在并且内容完整：

1. 读取 `research/<tech-name>/01-discovery.md`——技术发现报告，包含技术概览、核心特性、生态系统等基础信息。
2. 读取 `research/<tech-name>/02-analysis.md`——技术分析报告，包含深度技术分析、架构设计、性能评估、优劣势对比等详细内容。

如果上述文件不存在或内容不完整，应当先提示用户运行前置的调研技能（tech-research、tech-analysis）来完成这些报告，然后再执行本技能。从两份报告中提取所有核心概念、关键知识点、技术架构、代码示例以及注意事项，作为教程内容编排的基础素材。

## 执行流程

按照以下步骤严格执行教程生成流程：

### 第一步：读取并分析前置报告

使用 Read 工具依次读取 `research/<tech-name>/01-discovery.md` 和 `research/<tech-name>/02-analysis.md`。仔细分析两份报告的内容，提取以下关键信息：

- 技术的核心概念列表和概念之间的依赖关系
- 关键特性和功能点，按学习难度从低到高排序
- 技术架构和设计理念
- 已有的代码片段和示例
- 常见陷阱、已知问题和最佳实践
- 生态系统中的相关工具和库

### 第二步：设计教程结构

根据提取的知识点，设计教程的整体结构。教程必须按照以下六大模块组织：

1. **环境搭建指南**——系统要求、安装步骤、验证安装
2. **入门篇**——最基础的核心概念，每个概念独立成节，难度递进
3. **进阶篇**——在入门篇基础上展开的关键特性，引入更复杂的用法
4. **高级篇**——高级用法、性能优化、最佳实践
5. **实战项目**——综合运用前述知识点的完整项目
6. **常见问题与排查指南**——错误排查表、调试技巧、学习路线推荐

在设计过程中，确保每个知识点严格建立在前一个知识点之上，不允许出现知识跳跃。如果发现前置报告中的某个概念需要额外的背景知识，在教程中先行补充。

### 第三步：编写各模块内容

为每个知识点编写以下四个部分：

- **概念讲解**：用通俗易懂的语言解释概念原理，说明为什么需要这个概念、解决了什么问题、与其他概念的关系
- **代码示例**：编写完整可运行的代码，包含所有必要的 import 语句和依赖声明，禁止使用伪代码或省略号
- **执行结果**：标注每个代码示例的预期输出，使读者能够对照验证
- **练习题**：为每节设计 1-3 道练习题，帮助读者巩固所学知识

### 第四步：设计实战项目

在教程的第四部分，设计一个贯穿性的综合实战项目。项目需求必须描述清晰，架构设计合理。项目代码必须完整可运行，并且必须综合运用教程中至少 3 个不同的知识点。在代码之后，逐段解析关键实现逻辑。最后提供 2-3 个扩展挑战，鼓励读者在已有代码基础上进行深入探索。

### 第五步：编写常见问题和调试指南

根据前置报告中提到的问题以及技术社区常见痛点，编写常见错误排查表。表格格式为「错误信息 | 原因 | 解决方案」。同时编写 2-3 条实用的调试技巧，帮助读者在遇到问题时快速定位原因。

### 第六步：编写学习路线推荐

整理官方文档的推荐阅读顺序，标注每章的重点内容。推荐 2-3 个高质量的进阶学习资源（书籍、课程、开源项目等），帮助读者在完成本教程后继续深入学习。

### 第七步：格式化输出

按照 `references/output-template.md` 模板格式化最终输出。将模板中的占位符替换为实际内容，保持模板的结构和层级不变。确保所有 Markdown 语法正确，代码块标注了正确的语言类型。

### 第八步：保存输出文件

将最终生成的教程保存到 `research/<tech-name>/03-tutorial.md`。确认文件写入成功后，向用户报告完成状态和文件路径。

## 质量约束

生成教程时必须严格遵守以下五条硬性规则：

1. **循序渐进，严禁跳跃**。教程的每一节必须严格建立在前一节的基础之上。新概念只能在已有概念被充分讲解之后引入。如果某个概念需要前置知识，必须先补充前置知识的讲解，不允许跳过任何中间步骤。

2. **代码示例必须完整可运行**。所有代码示例必须是完整的、可以直接复制执行的代码。禁止使用伪代码、省略号、`// ...此处省略...` 等占位符。每个代码示例必须包含所有必要的 import 语句、依赖声明和完整实现。代码中的注释用中文编写，解释关键逻辑。

3. **每个代码示例必须标注预期输出**。在代码示例之后，使用独立的代码块标注该代码运行后的预期输出结果。预期输出应当准确反映代码的实际执行结果，使读者能够对照验证自己的执行是否正确。

4. **标注常见坑点和避坑方法**。在进阶篇和高级篇的每个知识点讲解中，必须包含「注意事项」小节，列出该知识点相关的常见陷阱、容易犯的错误以及对应的避坑方法。注意事项的内容应当来自前置报告中的分析以及技术社区的常见问题。

5. **实战项目必须综合运用至少 3 个知识点**。第四部分的综合实战项目必须覆盖教程前述内容中至少 3 个不同的知识点。在「代码解析」部分，必须明确指出每一处运用了哪个知识点，帮助读者将项目代码与所学知识对应起来。

## 输出规范

- **输出文件路径**：`research/<tech-name>/03-tutorial.md`
- **输出格式**：严格遵循 `references/output-template.md` 中定义的 Markdown 模板格式
- **语言**：教程正文使用中文撰写，代码注释使用中文
- **代码块**：使用正确的语言标识（如 `bash`、`python`、`javascript` 等）
- **表格**：常见错误排查表使用标准 Markdown 表格格式
- **层级**：保持模板中定义的标题层级结构，不得随意增减层级
